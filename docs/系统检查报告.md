# 系统检查报告

## 检查时间
最近一次：代码静态检查 + 导入与配置校验

## 已修复问题

1. **`perception/crypto_connector.py`**  
   - 问题：`watch_orderbook(symbol, limit=20, callback)` 中带默认参数 `limit` 出现在无默认参数 `callback` 之前，导致 SyntaxError。  
   - 修复：改为 `watch_orderbook(symbol, callback, limit=20)`。

2. **`requirements.txt`**  
   - 问题：回测依赖写为 `backtracking`，应为 `backtesting`。  
   - 修复：已改为 `backtesting>=0.2.0`。

3. **`evolution/__init__.py`**  
   - 问题：直接导入 `backtest_engine`、`strategy_generator`、`genetic_optimizer`，在未安装 polars 等依赖时导致 ModuleNotFoundError，影响 AetherLife 启动。  
   - 修复：仅强制导出 `EvolutionEngine`；其余模块改为 try/except 可选导入，缺依赖时不影响主流程。

4. **AetherLife 启动时记忆恢复**  
   - 问题：`load_from_redis()` 从未在启动时调用。  
   - 修复：在 `core/life.py` 的 `run()` 中，在进入主循环前调用 `await self._memory.load_from_redis()`（若存在）。

## 校验结果

- **主系统**：`trading_bot`、`data_fetcher`、`strategies.factory`、`exchange_client`、`risk_manager`、`utils.config` 导入与 `validate_config(DEFAULT_CONFIG)` 通过。
- **AetherLife**：`AetherLife`、`AetherLifeConfig`、`Orchestrator`、`RiskGuard`、`MemoryStore`、`EvolutionEngine` 导入通过。
- **配置**：`AetherLifeConfig.from_dict({})` 与 `from_dict({"symbol": "ETHUSDT"})` 行为正确。
- **Lint**：当前无 linter 报错。

## 运行前注意

- 从 **src** 目录运行：`cd src && python -m aetherlife.run` 或 `python -m aetherlife.run`（当前目录为 src）。
- 从 **项目根** 运行：`python src/aetherlife/run.py`，会加载 `configs/aetherlife.json`（若存在）。
- 若使用 Redis 记忆：需安装 `redis` 并配置 `REDIS_URL`；启动时会自动尝试 `load_from_redis()`。
- 进化层高级模块（BacktestEngine、StrategyGenerator、GeneticOptimizer）依赖 polars 等，未安装时仅核心 `EvolutionEngine` 可用，主循环与进化调度正常。
