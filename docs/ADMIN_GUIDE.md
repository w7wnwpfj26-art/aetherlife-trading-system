# 后台管理系统使用文档

## 🎯 功能概述

后台管理系统提供了一个完整的Web界面，用于配置和管理AI合约交易系统。

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动后台管理系统

```bash
python start_admin.py
```

### 3. 访问管理界面

在浏览器中打开：`http://127.0.0.1:8080/admin`

## 📋 功能说明

### 🔑 API配置

**功能：**
- 选择交易所（Binance/OKX）
- 配置API Key和Secret Key
- 测试交易所连接
- 测试API密钥有效性
- 切换测试网/主网

**如何获取API密钥：**

**Binance:**
1. 登录币安账号
2. 用户中心 → API管理 → 创建API
3. 权限设置：读取信息 + 现货和杠杆交易（不要开启提币权限）

**OKX:**
1. 登录OKX账号
2. 个人中心 → API → 创建API密钥
3. 权限设置：读取 + 交易（不要开启提币权限）

**⚠️ 安全提示：**
- API密钥使用AES加密存储在本地
- 不会上传到任何服务器
- 建议不要授予提币权限
- 定期更换API密钥

### 📊 策略配置

**支持的策略：**

1. **突破策略** - 趋势跟随
   - 回看周期：计算突破的K线数量
   - 突破阈值：价格突破的最小百分比
   - ATR倍数：用于计算止损

2. **网格策略** - 震荡市场
   - 网格数量：分成多少层网格
   - 网格间距：每层之间的价格间距
   - 基础价格：网格中心价格

3. **均线交叉** - 经典策略
   - 快线周期：短期均线
   - 慢线周期：长期均线

4. **RSI策略** - 超买超卖
   - RSI周期：计算RSI的K线数量
   - 超卖线：低于此值做多
   - 超买线：高于此值做空

5. **成交量策略** - 量价分析
   - 成交量均线：计算平均成交量
   - 放量倍数：成交量放大倍数

**其他配置：**
- 交易对：支持多个交易对（用逗号分隔）
- 时间周期：1m, 5m, 15m, 1h, 4h, 1d
- 杠杆倍数：1-125倍（建议不超过10倍）

### 🛡️ 风控设置

**关键参数：**

- **最大仓位比例**：单笔交易最多使用账户资金的百分比
  - 建议：10%
  - 说明：控制单笔交易风险

- **止损比例**：从入场价格计算的最大亏损百分比
  - 建议：1-3%
  - 说明：亏损达到此比例自动平仓

- **止盈比例**：从入场价格计算的目标盈利百分比
  - 建议：止损的2-3倍（2-9%）
  - 说明：盈利达到此比例自动平仓

- **每日最大亏损**：单日累计亏损达到此比例后停止交易
  - 建议：5%
  - 说明：防止连续亏损导致爆仓

**风控原则：**
- 止损必须设置，不要扛单
- 止盈应该是止损的2-3倍
- 单笔仓位不超过总资金的10%
- 每日亏损不超过总资金的5%

### 🧠 AI增强

**功能开关：**

1. **多Agent协作**
   - 7个AI Agent协同分析
   - 权重投票决策
   - 提高决策准确性

2. **机器学习预测**
   - 自动特征工程
   - 实时模型训练
   - 预测价格走势

3. **市场情绪分析**
   - 社交媒体情绪
   - 新闻情感分析
   - 恐慌/贪婪指数

4. **自动复利管理**
   - 利润自动复投
   - 仓位再平衡
   - 最大化资金效率

**注意：**
- AI功能需要更多计算资源
- 可能影响系统性能
- 建议根据实际情况开启

### ⚙️ 系统设置

**配置管理：**
- 保存所有配置：将当前配置保存到文件
- 重新加载配置：从文件加载配置
- 恢复默认设置：重置为系统默认值
- 导出配置文件：导出JSON格式配置

**Bot控制：**
- 启动交易：开始自动交易
- 停止交易：停止自动交易

**配置文件位置：**
```
configs/
  ├── config.json      # 普通配置
  ├── secure.enc       # 加密的API密钥
  └── .key             # 加密密钥（自动生成）
```

## 🔒 安全性

### 加密存储

- API密钥使用Fernet对称加密
- 加密密钥存储在本地`.key`文件
- 文件权限设置为600（仅所有者可读写）

### 数据隔离

- 敏感信息与普通配置分离存储
- 密钥文件与配置文件分离
- 导出配置时可选是否包含敏感信息

### 最佳实践

1. 不要共享`.key`文件
2. 定期更换API密钥
3. 不要授予提币权限
4. 先在测试网验证
5. 定期备份配置文件

## 📝 使用流程

### 首次使用

1. **配置API**
   - 填写交易所API密钥
   - 开启测试网模式
   - 测试连接是否成功

2. **配置策略**
   - 选择适合的策略
   - 设置策略参数
   - 选择交易对和时间周期

3. **设置风控**
   - 设置止损止盈
   - 配置仓位管理
   - 设置每日最大亏损

4. **保存配置**
   - 在"系统设置"页面保存配置
   - 确认配置已保存成功

5. **启动测试**
   - 在测试网运行至少1周
   - 观察策略表现
   - 调整参数优化

6. **实盘运行**
   - 关闭测试网模式
   - 填写实盘API密钥
   - 从小资金开始

### 日常使用

1. 登录管理后台
2. 查看Bot运行状态
3. 检查交易表现
4. 根据需要调整参数
5. 定期导出配置备份

## ⚠️ 注意事项

### 风险提示

1. **高杠杆高风险**
   - 合约交易可能亏光本金
   - 建议杠杆不超过10倍
   - 永远不要投入无法承受损失的资金

2. **先测试后实盘**
   - 在测试网运行至少1个月
   - 确保策略稳定盈利
   - 了解所有功能和风险

3. **AI不是万能的**
   - AI也会出错
   - 市场有不可预测性
   - 不要盲目依赖AI

4. **持续监控**
   - 定期检查系统运行状态
   - 关注市场重大变化
   - 及时调整策略参数

### 技术要求

- Python 3.8+
- 稳定的网络连接
- 足够的系统资源（AI功能）
- 可靠的服务器（24小时运行）

## 🆘 常见问题

### Q: API连接失败？
A: 检查以下几点：
- API密钥是否正确
- 网络是否正常
- 交易所是否维护
- 测试网/主网是否选对

### Q: 配置保存失败？
A: 可能原因：
- 磁盘空间不足
- 文件权限问题
- 配置格式错误

### Q: Bot启动失败？
A: 检查：
- 配置是否完整
- API密钥是否有效
- 是否有足够资金
- 日志中的错误信息

### Q: 如何备份配置？
A: 两种方式：
1. 使用"导出配置"功能
2. 直接复制`configs/`文件夹
